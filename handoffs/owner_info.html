Tab 8 — Owner Info

Fill Owner Info

Universal — fully payload-driven, including the Owner Name field. No builder branch — the bookmarklet writes whatever the session provides, or leaves Matrix's pre-populated value alone if the key is omitted.

Fill Owner Info
To install:
1. Make bookmarks bar visible — Cmd+Shift+B
2. Drag the blue button to the bookmarks bar
3. Place in the Matrix — New, Matrix — Tax ID, or Matrix — Copy folder

To use:
1. Copy the full JSON payload to clipboard
2. Navigate to the Owner Info tab in Matrix
3. Click the bookmark — fields populate from the payload
⚠️ Owner Name (Input_118) — unconditional write, no taxid skip. Unlike most fields on this tab, the bookmarklet writes Owner Name whenever owner_name is present in the payload — regardless of path. This matches Lennar's force-overwrite behavior, but the bookmarklet has no idea it's Lennar; it's just following the payload. For a standard taxid listing where Matrix's tax-record owner name should be left alone, the session must omit the owner_name key entirely from the payload.

⚠️ Open design question carried from the unification session: whether standard listings ever need an Owner Name override resolved the same way Lennar's force-overwrite works, or whether this field stays Lennar-only in practice. Not resolved here — flagged for the session to decide when it's relevant.

"owner": { "owner_name": "Lennar", // omit this key entirely to leave Matrix's // pre-populated taxid value untouched "occupant_name": "None", "occupied_by": "V", // e.g. "V" (Vacant) "owner_agent": "0", "agent_related_to_seller": "0", "owned_by": "02", // checkbox value — e.g. "02" (Corporate) "possession": "01" // checkbox value — e.g. "01" (At Closing) }
